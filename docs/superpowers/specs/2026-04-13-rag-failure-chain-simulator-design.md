# 第25章 RAG 故障链模拟器设计稿

## 背景

第 25 章当前已经有 `RagAccuracyDemo`，但它主要在对比：

- 混合检索
- 纯语义检索

这能说明“关键词召回为什么重要”，但还没有把整章真正的五类故障链串起来：

- 分块错误
- Embedding 失配
- 关键词缺失
- Prompt 边界失控
- 元数据冲突未治理

因此读者会看到一个局部案例，却不容易形成“RAG 出错时该按什么顺序排查”的完整心智模型。

## 目标

升级现有 `RagAccuracyDemo`，让它变成“RAG 五故障链模拟器”，围绕章节主链提供五个可切换阶段：

- 召回单元
- 相似度判断
- 关键词召回
- 回答边界
- 冲突治理

成功标准：

- 顶部可切换五个故障阶段
- 每个阶段有独立的错误现象和修复提示
- 第三阶段继续复用现有“混合检索 vs 纯语义检索”对比
- 右侧始终能看到“当前错在哪里、应该怎么修”
- 组件仍使用 `RagAccuracyDemo` 名称，不需要正文改组件标签

## 范围

本次只改第 25 章和现有 `RagAccuracyDemo.vue`。

包含：

- 升级 `RagAccuracyDemo.vue`
- 在正文中补一句操作提示
- 在章节体验校验中增加第 25 章阶段切换回归要求

不包含：

- 不新增平行的大型 RAG 组件
- 不引入真实向量检索或 rerank 逻辑
- 不重写整章正文

## 方案选择

### 方案 A：升级现有 `RagAccuracyDemo`

做法：

- 保留组件名
- 在组件内部新增五阶段状态
- 把现有场景对比嵌入“关键词召回”阶段

优点：

- 文档改动最小
- 复用现有数据和样式基础
- 章节主链能直接落到一个组件里

缺点：

- 组件内部逻辑会比原来更复杂

### 方案 B：新增 `RagFailureChainSimulator`

做法：

- 新建章节级组件
- 保留旧的 `RagAccuracyDemo`

优点：

- 新旧职责分离

缺点：

- 会出现两个 RAG 演示组件并存
- 第 25 章收益不如直接升级现有组件

### 推荐方案

采用方案 A。

原因：

- 第 25 章缺的是“故障排查顺序”，不是缺一个新容器
- 保留现有组件名，接入成本最低

## 交互结构

### 顶部阶段条

五个阶段：

- 召回单元
- 相似度判断
- 关键词召回
- 回答边界
- 冲突治理

切换阶段时：

- 不重置组件整体
- 主视图区和右侧说明一起切换

### 阶段一：召回单元

重点展示：

- 坏分块：过大或过小导致知识单元失真
- 好分块：按段落/规则边界切开

目标：

- 让用户理解 RAG 第一性问题不是“模型会不会答”，而是“切出来的单元对不对”

### 阶段二：相似度判断

重点展示：

- 同一查询在不同语义空间下的匹配差异
- 中文语料和错误 embedding 的失配

目标：

- 让用户明白“相似”不是抽象概念，而是 embedding 模型定义出来的

### 阶段三：关键词召回

重点展示：

- 继续复用当前混合检索 vs 纯语义检索对比
- 保留 precision / recall 和答案对照

目标：

- 解释为什么版本号、数字、专有名词必须补关键词能力

### 阶段四：回答边界

重点展示：

- 宽松 prompt 下的自由发挥
- 严格 prompt 下的边界收口

目标：

- 让用户看到“检索到了，不代表答案就安全”

### 阶段五：冲突治理

重点展示：

- 多份文档来源冲突
- 元数据排序规则
- 最终选择哪一份作为有效来源

目标：

- 让用户理解没有 metadata 治理时，模型只能拍脑袋决定谁优先

## 视觉结构

建议结构：

- 顶部：标题 + 五阶段按钮
- 左侧：当前阶段的检索/文档视图
- 中部：当前阶段的结果或对照区
- 右侧：当前故障说明、修复建议、排查顺序提示

整体仍保持“工程诊断面板”风格，而不是炫技动画。

## 数据设计

保留现有 `scenarios` prop，继续用于第三阶段。

其他阶段演示数据直接内置到组件内：

- 分块对照文本
- embedding 匹配样例
- prompt 对照答案
- metadata 冲突案例

原则：

- 不改全站 `RagScenario` schema
- 不为了演示新增复杂数据层

## 章节接入方式

正文里继续使用 `<RagAccuracyDemo />`。

只在组件前补一句：

“按顺序切换五个故障阶段，观察 RAG 是在哪一层开始答偏的。”

## 验证方案

至少验证：

- `bun run typecheck`
- `bun run check:chapter-experience`
- `bun run build`

章节体验校验新增要求：

- `RagAccuracyDemo.vue` 必须包含阶段切换处理函数
- 必须包含五个阶段标识：
  - 召回单元
  - 相似度判断
  - 关键词召回
  - 回答边界
  - 冲突治理

## 风险与控制

### 风险 1：组件信息量过大

控制：

- 每个阶段只讲一个故障根因
- 不在同一屏同时展示五种问题

### 风险 2：现有混合检索对比被埋没

控制：

- 保留它作为第三阶段核心视图
- 不删现有 precision / recall 指标

### 风险 3：阶段切换后像五个不相关的小 demo

控制：

- 右侧始终强调“这是 RAG 供应链上的第几层”
- 用同一套排查语言串起来
