# Agent 面试题专区设计稿

## 背景

当前站点已经有：

- 主线章节内容
- 实践篇
- 中级篇
- 若干独立专栏

但还缺一个面向“面试准备”的独立入口。

这个需求是个真实问题，不是功能堆砌：

- 读者学完 Agent 相关内容后，往往会进入“怎么整理成面试表达”的阶段
- 现有内容更偏学习路径，不是按面试能力分类组织
- 如果把面试题直接塞进现有章节，会污染章节语义，也会让导航结构变混

因此更合适的方式，是新增一个独立模块：`Agent 面试题专区`。

## 目标

新增一个独立导航模块，用来系统整理 Agent 面试常见问题，第一版聚焦：

- 独立导航入口
- 一个总览页
- 7 个能力分类子页
- 每页提供高频题、回答思路、追问方向

成功标准：

- 顶层导航可以直接进入 `Agent 面试题专区`
- 侧边栏具备独立结构，不混入 `practice/` 或 `intermediate/`
- 总览页能完成分流，而不是把所有题目堆在一页
- 分类页结构统一，便于后续持续追加内容

## 方案选择

### 方案 A：单页专题

只做一个 `/interview/` 页面，把所有分类和题目都堆在一页。

优点：

- 上线最快
- 改动最小

缺点：

- 页面会很快变长
- 后续扩展时结构容易失控
- 不适合做独立模块

### 方案 B：独立模块

新增一个独立导航入口，配置单独目录、总览页和分类页，并提供专区 sidebar。

优点：

- 信息结构最清晰
- 不破坏现有章节编号体系
- 后续扩展成本低

缺点：

- 需要新增导航和 sidebar 配置

### 方案 C：发现中心式入口

保留独立导航，但入口页更像发现中心，重点放在导流和卡片分发。

优点：

- 首页感更强

缺点：

- 第一版偏重
- 在内容数量还不够多时收益不高

### 推荐方案

采用方案 B。

原因：

- 这是个真实问题，值得独立建模
- 数据结构先行，比视觉包装更重要
- 不引入额外复杂度，也不破坏现有内容边界

## 信息架构

新增顶层导航：

- `面试题专区` -> `/interview/`

目录结构建议：

- `docs/interview/index.md`
- `docs/interview/fundamentals/index.md`
- `docs/interview/tools/index.md`
- `docs/interview/memory/index.md`
- `docs/interview/planning/index.md`
- `docs/interview/rag/index.md`
- `docs/interview/multi-agent/index.md`
- `docs/interview/engineering/index.md`

侧边栏建议作为独立小册子组织：

- 课程介绍 / 总览
- 基础概念
- 工具调用
- 记忆
- 规划
- RAG
- Multi-Agent
- 工程化

这样处理有三个好处：

- 不污染现有主线章节结构
- 分类稳定，后续加题只需追加内容
- 用户进入后认知负担低

## 页面设计

### `/interview/` 总览页

总览页负责分流，不负责堆题。

建议结构：

1. 说明这个专区解决什么问题
2. 按能力分类展示 7 个入口
3. 每个入口包含：
   - 主要考点
   - 适合层级
   - 进入分类页链接

总览页需要明确传达：

- 这是一个“按能力准备 Agent 面试”的入口
- 不是随机题库
- 不是照本宣科的八股页

### 分类子页

所有分类页统一采用同一套结构：

1. 这一类为什么常考
2. 高频题列表
3. 每道题固定 3 段
   - `题目`
   - `回答思路`
   - `追问方向`
4. 页尾补“书内关联阅读”

统一结构的价值：

- 后续加题时不需要为每页重新发明格式
- 降低特殊情况
- 用户形成稳定阅读预期

## 能力分类

第一版按能力分类组织：

- 基础概念
- 工具调用
- 记忆
- 规划
- RAG
- Multi-Agent
- 工程化

不按轮次分类，也不按题型分类。

原因：

- 与现有站点的学习路径更一致
- 更符合 Agent 面试真实考察方式
- 更利于和书内章节做映射

## 第一版内容边界

第一版采用轻量版，不做大题库。

建议规模：

- 每个分类先放 5 题
- 共 35 题

每题只写到“面试可口述”的深度，不展开成长文答案。

每题内容固定包含：

- `题目`：面试官可能如何问
- `回答思路`：答题骨架，不追求长篇铺陈
- `追问方向`：帮助读者知道下一层会被怎么深挖

这样处理的原因：

- 先验证结构和入口，而不是一上来做成重型题库
- 先覆盖完整能力面，再逐步加深每个分类
- 后续从 5 题扩到 8 题或更多时，不需要重做信息架构

## 范围

本次设计覆盖：

- 顶层导航入口
- 专区 sidebar
- 总览页
- 7 个分类子页
- 第一版内容结构规范

本次不包含：

- 标签筛选
- 搜索型题库系统
- 随机抽题
- 折叠答案交互
- 评分系统
- 面试模拟器

## 实施原则

实现时必须遵守：

- 最小修改
- 不改动无关章节结构
- 不引入额外复杂交互
- 先建立稳定数据结构，再考虑视觉增强

## 验证方式

至少验证：

- 顶层导航可进入 `/interview/`
- sidebar 能正确展示专区结构
- 总览页和 7 个分类页都可访问
- 页面间跳转正常
- `bun run build` 可通过
