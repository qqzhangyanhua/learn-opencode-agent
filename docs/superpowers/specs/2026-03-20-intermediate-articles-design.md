## Section 3：单篇文章统一模板

中级篇中的每一章，都统一改写为电子书章节体例，而不是保留公众号文章风格。

### 固定结构

每篇文章建议采用统一模板：

1. 标题 + 一句话摘要
2. 对应路径 / 前置阅读 / 学习目标
3. 这篇解决什么问题
4. 为什么这个问题在真实 Agent 系统里重要
5. 核心概念与主链路
6. 源码映射
7. 教学代码示例
8. 工程上的常见误区
9. 延伸阅读与回链

### 改写原则

需要保留的内容：

- 参考文章的核心问题意识
- 原有的案例顺序
- 关键对比、关键坑点和关键工程判断
- 对应的教学代码示例

需要移除或弱化的内容：

- 公众号封面提示词
- 过强的自媒体导流文案
- 与整本书语气不一致的口语化转场
- 不适合电子书连续阅读的情绪化包装

需要新增的内容：

- 前置阅读
- 与本书已有章节的关系说明
- 源码映射
- 延伸阅读与回链

### 风格要求

统一采用当前电子书已经建立的书写方式：

- 章节化结构
- 明确的导读与学习目标
- 从“入口文件 / 主链路 / 容易误解的点”切入
- 强调工程判断，而不是只堆概念

## Section 4：源码映射与教学示例的双映射策略

用户明确要求“保留源码映射”，因此中级篇不能只是泛化的专题文章集合，必须继续锚定现有 OpenCode 主线。

### 双映射结构

每篇文章都固定保留两类映射：

#### 1. OpenCode 源码映射

用于告诉读者：这一章讨论的工程问题，在 OpenCode 当前仓库里应该回看哪些入口文件。

展示方式：

- 继续复用现有 `SourceSnapshotCard.vue`
- 标注仓库、分支、commit、验证日期和关键入口文件

#### 2. 教学代码示例映射

用于告诉读者：如果想看一个更聚焦、更容易运行的教学示例，应该去当前仓库里的哪个目录。

展示方式：

- 在正文“教学代码示例”一节列出对应目录和文件
- 明确标注“这是教学示例，不是 OpenCode 原仓实现”

### 这样设计的原因

如果不做这层区分，读者很容易误解：

- 误以为 Python 示例就是 OpenCode 原始实现
- 误以为章节讨论的是脱离当前源码主线的泛 Agent 文章

双映射的作用，就是同时保住两条线：

- **源码理解线**：继续服务当前电子书的 OpenCode 源码阅读目标
- **教学实验线**：提供可运行、可验证、可快速对照的简化代码

## Section 5：代码展示与折叠规则

用户要求：

- 如果文章有代码示范，希望展示完整代码
- 如果代码太多，则引导用户去目录查看

因此，中级篇需要一套明确的代码展示策略，避免每篇自由发挥。

### 三档展示规则

#### 档位 A：短代码

适用范围：

- 20-60 行左右
- 读者一屏内能读完
- 对理解正文主论点有直接帮助

展示方式：

- 直接内嵌正文代码块

#### 档位 B：中等长度完整代码

适用范围：

- 单文件
- 约 60-200 行
- 读者有必要看到完整实现，但不适合默认展开占满页面

展示方式：

- 正文使用折叠块默认收起
- 用户点击后可查看完整代码

推荐优先采用 VitePress 原生容器能力，例如：

````md
::: details 查看完整代码
```python
# 完整示例
```
:::
````

这样做的原因：

- 不需要为了首批中级篇先引入新的重型代码组件
- 与现有 VitePress 内容体系兼容
- 成本最低，后续如果需要再抽象成统一组件

#### 档位 C：长代码或多文件代码

适用范围：

- 超过约 200 行
- 由多个文件共同组成
- 代码本身更像一个小型示例目录，而不是单个片段

展示方式：

- 正文只保留关键流程、核心函数或核心片段
- 在“教学代码示例”一节链接到仓库内的示例目录页
- 明确提示“完整代码请查看本章示例目录”

### 总体规则

建议将以下准则固化：

- 单文件且长度适中：正文允许折叠查看完整代码
- 超长代码或多文件结构：正文只放关键片段，完整代码下沉到示例目录

这样可以同时满足两个目标：

- 读者在正文里尽量能看到完整实现
- 文档页不会退化为纯代码仓库浏览器

- 使用三档代码展示规则
- 长代码下沉到示例目录

- 固定单篇模板
- 统一导读、前置阅读、学习目标、源码映射、延伸阅读结构

#### 风险 4：中级篇入口加重整体导航复杂度

- 首页只增加一个明确入口

## 最终建议

推荐采用以下落地方案：

1. 新增一个独立的“第六部分：中级专题与工程进阶”
2. 首批接入 8 篇热门中级专题
3. 所有文章统一改写为电子书风格，而不是保留公众号体
4. 每篇固定保留 OpenCode 源码映射与教学示例映射
5. 代码展示采用“三档策略”：短代码正文展示、中等代码折叠展示、长代码目录下沉
6. 外部代码示例迁入当前仓库，建立稳定的目录引用

这套方案能在不破坏现有主书结构的前提下，快速建立一个具有吸引力、可持续扩展、且与 OpenCode 主线保持一致的中级篇模块。
