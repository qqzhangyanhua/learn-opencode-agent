# Phase 2 首页与导航产品化入口设计文档

**日期**: 2026-03-22  
**范围**: 首页、顶层导航、侧边栏、阅读地图、实践页、中级篇入口层  
**依赖**: Phase 1 已完成的 `content-index.data.ts`、`learning-paths.data.ts`、`SectionRoleGrid.vue`、`LearningPath.vue`

---

## 背景

Phase 1 已经把“学习路径、板块定位、实践阶段摘要”收敛成统一数据层，用户现在已经可以访问 `/learning-paths/`，也能在首页、实践页和中级篇看到一致的板块定位。

但当前站点仍然存在两个产品层问题：

1. **首页还是偏文档首页**  
   用户能看到很多信息，但首屏仍然不够像“产品入口”，还缺更明确的角色分流、起步建议和下一步选择。

2. **导航仍然偏章节树而不是产品入口**  
   虽然已经加入“学习路径”入口，但顶层导航与根侧边栏仍然以章节编排为主，不够突出“新用户该去哪、不同目标的人该选哪条路”。

Phase 2 的目标不是重新定义学习路径，而是把 Phase 1 已完成的数据真源，真正变成首页和导航层的产品体验。

---

## 成功标准

这一阶段完成后，下面三件事必须为真：

1. 新用户打开首页后，能在 30 秒内识别至少一条适合自己的起步路线。
2. 用户不理解完整章节树，也能从导航快速进入学习路径、实践入口和中级专题入口。
3. 用户在首页之外的关键入口页，仍然知道“我现在在哪个板块、下一步该去哪”。

---

## 方案比较

### 方案 A：只重排首页文案

只修改 `docs/index.md` 与少量导航文案，不新增入口组件。

优点：
- 实现快
- 改动小

缺点：
- 首页会更整齐，但仍像“优化后的文档页”
- 很难真正满足 `IA-01` 和 `HOME-01`

### 方案 B：数据驱动的产品化入口层

基于 Phase 1 数据层，新增首页分流区、强化导航入口、在关键 landing page 增加“当前位置 + 下一步”组件。

优点：
- 与当前目标最一致
- 最大化复用 Phase 1 数据层
- 后续 Phase 3 / Phase 4 可以继续沿用

缺点：
- 需要调整首页结构、导航配置和少量主题组件

### 方案 C：视觉先行的大改版

优先大幅重构首页视觉和导航视觉系统，再回头补分流逻辑。

优点：
- 视觉变化最大

缺点：
- 很容易稀释“清晰起步入口”的核心目标
- 风险高，容易超出 Phase 2 范围

**推荐方案：B。**

理由：当前真正缺的是“入口清晰度”和“产品化分流”，不是单纯视觉换皮。Phase 1 已经具备完整数据底座，Phase 2 最应该做的是把这些数据变成用户真的能用的首页和导航入口。

---

## 设计结论

### 1. 首页要从“说明页”升级成“起步页”

首页应突出三层信息：

1. **我适合从哪里开始**
2. **这站里三大板块分别负责什么**
3. **我接下来可以走哪条路线**

具体表现为：

- Hero CTA 缩减并强化主路径
- 新增面向不同目标用户的起步分流卡片
- 保留 `LearningPath`，但它承担“路径选择器”，不再承担“解释首页结构”的职责
- 首页下半部分继续保留系统主链路和技术栈，但不抢首屏入口注意力

### 2. 导航要从“章节入口”升级成“产品入口”

顶层导航应优先暴露：

- 首页
- 学习路径
- 实践篇
- 中级篇
- 阅读地图 / 补充入口

侧边栏需要把“从哪开始”和“你正在看哪个板块”表达清楚，不能只给出长章节树。

### 3. 关键 landing page 都要告诉用户“你在哪、下一步去哪”

重点不是在每一章都加复杂导航，那是 Phase 3 的任务。Phase 2 先处理入口页：

- `/`
- `/learning-paths/`
- `/practice/`
- `/intermediate/`
- `/reading-map`

这些页面需要统一的“板块定位 + 推荐下一步”表达。

---

## Phase 2 拆分

### 02-01 首页首屏与角色分流区

目标：
- 把首页变成真正的产品入口
- 让新用户快速完成自我判断和起步选择

### 02-02 顶层导航、侧边栏与板块入口

目标：
- 让用户无需理解章节树，也能进入正确入口
- 让“学习路径”成为真正的一等入口

### 02-03 关键入口页的当前位置与下一步提示

目标：
- 在首页之外的关键页继续维持清晰的信息架构
- 降低用户从首页跳出后的迷路概率

---

## 风险与约束

1. Phase 2 不能重新引入手写路径数组或手写 phase 数组，必须复用 Phase 1 数据层。
2. 不做章节级模板改造，那属于 Phase 3。
3. 不做登录、进度系统、搜索增强、AI 助教等超范围能力。
4. 视觉可以更产品化，但不能偏离当前站点的 teal 品牌体系和现有 VitePress 结构。

## 验证方式

自动验证：
- `bun run build:strict`

人工验证：
- 打开 `/`，确认新用户能快速看见至少一条起步路线
- 打开顶层导航，确认“学习路径 / 实践篇 / 中级篇”入口清晰
- 打开 `/practice/`、`/intermediate/`、`/reading-map`，确认都有“当前定位 + 下一步入口”
