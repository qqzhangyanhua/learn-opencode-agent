# 93 术语表

遇到不认识的词，来这里查。每个词只说最核心的含义，不展开细节。

---

## A

**AgentTool**
一种特殊工具，执行时会启动一个独立的子 Agent 来完成任务，结果再回流主会话。对应第 14 章。

**AsyncGenerator（异步生成器）**
JavaScript/Python 里的一种函数类型，可以在运行过程中"吐出"多个中间值，而不是等运行完才返回一个结果。Claude Code 用它实现流式输出和主循环。

**allow / deny / ask**
权限系统的三种判定结果。`allow` 直接执行，`deny` 直接拒绝，`ask` 暂停等用户确认。对应第 10 章。

---

## B

**blockingError**
stop hook 产出的一种结果：这轮 assistant 输出不合法，把错误追加进消息历史，强制再给模型一轮修正机会。不是终止，而是"要求重试"。

**bundled skill**
随 Claude Code 一起打包分发的内置 Skill，第一次调用时会被提取到本地目录，供工具通过文件路径读取。

---

## C

**compact（上下文压缩）**
当消息历史太长、接近 API 上下文限制时，把历史摘要化，腾出空间让主循环继续工作。Claude Code 有多层压缩策略（microcompact、autocompact 等）。

**contextModifier**
工具返回值里的一个可选字段，允许工具在执行成功后修改 `ToolUseContext`（比如更新已加载的文件状态）。

---

## F

**finish_reason**
OpenAI API 里模型返回时带的一个字段，说明"模型为什么停止输出"。`tool_calls` 表示模型要调用工具，`stop` 表示模型认为任务完成。

**fork skill**
frontmatter 里写了 `context: fork` 的 Skill，执行时会在独立子 Agent 里运行，不污染主会话的消息历史。

**frontmatter**
Markdown 文件开头的 `---` 包裹的结构化元数据。Skill 用 frontmatter 声明允许使用的工具、执行模式、适用场景等。

---

## H

**Hook**
在特定时机自动触发的脚本或回调。Claude Code 里有多种 hook 类型：工具执行前/后（PreToolUse / PostToolUse）、模型采样后（PostSampling）、会话结束时（Stop hooks）等。

---

## I

**isConcurrencySafe**
工具定义里的一个方法，返回 true 表示该工具可以和其他工具并发执行（比如只读工具），返回 false 表示必须串行（比如有写操作的工具）。

---

## M

**MCP（Model Context Protocol）**
Anthropic 提出的标准协议，让 Agent 能通过统一接口连接外部服务（数据库、浏览器、第三方 API），而不用把每个外部服务的实现代码写进 Agent。对应第 07 章。

**MCP skill**
从 MCP server 动态加载的 Skill，运行时注入，和本地 Skill 走不同加载链。出于安全考虑，MCP skill 不能执行 inline shell 命令。

**messagesForQuery**
主循环每轮真正发给模型的消息窗口，不等于完整的消息历史——会经过压缩、裁剪、预算限制等处理后才送出。

---

## N

**needsFollowUp**
`query.ts` 里的一个布尔值，表示"这轮 assistant 输出里有 tool_use，主循环需要进入工具执行阶段"。由实际消息内容决定，不依赖 API 返回的 `stop_reason`。

---

## P

**Plugin**
能力扩展包，一个插件可以同时向系统注入 commands、skills、agents、hooks、MCP server 配置等多种能力。类比 VS Code 插件。对应第 09 章。

**PostSamplingHook**
模型输出结束后触发的观察型 hook，失败不阻断主流程，适合做日志、统计等后台操作。和 stop hooks 不同，它不能阻止主循环继续。

**preventContinuation**
stop hook 产出的一种结果：明确阻止主循环继续下一轮，返回 `stop_hook_prevented`。

---

## Q

**queryLoop()**
`query.ts` 里的核心函数，实现了 Agent 的主循环：准备消息窗口 → 调用模型 → 有工具就执行回填 → 判断是否继续 → 返回终止原因。

---

## S

**Skill**
以 Markdown 文件形式存储的方法论封装。用 frontmatter 声明元数据（允许的工具、执行模式），正文描述"做这件事的正确流程"。对应第 08 章。

**stop hooks**
主循环每轮结束前运行的治理型 hook，可以产出三种结果：summary（记录信息）、blockingError（强制重试）、preventContinuation（阻止继续）。

**stop_reason**
Anthropic API 里模型返回时带的字段（对应 OpenAI 的 `finish_reason`）。Claude Code 不完全依赖它，而是通过检查 assistant 输出里是否真的有 `tool_use` 来决定是否进入工具阶段。

---

## T

**terminal reason**
主循环返回时的终止原因，包括：`completed`（正常完成）、`max_turns`（超过最大轮次）、`aborted`（用户中断）、`hook_stopped`（hook 阻止继续）等。

**token budget**
系统侧对 token 用量的追踪，用来判断"这轮任务真的完成了吗"。如果模型说停止但 token 用量显示任务可能未完成，系统会注入一条 meta 消息让模型继续。

**tool_result**
把工具执行结果包装成消息的格式。在 OpenAI API 里是 `role: "tool"` 的消息，在 Anthropic API 里是 `content` 里 `type: "tool_result"` 的内容块。无论成功还是失败，工具执行都会生成 tool_result 回填给模型。

**tool_use**
模型发出的工具调用申请。在 OpenAI API 里是 `tool_calls` 字段，在 Anthropic API 里是 `content` 里 `type: "tool_use"` 的内容块。模型只负责"申请"，执行权在系统手里。

**ToolResult**
Claude Code 里工具 `call()` 方法的返回类型，包含：`data`（给模型看的结果）、`newMessages`（附加消息）、`contextModifier`（对运行时容器的修改）。

**ToolUseContext**
贯穿整个主循环的运行时容器，装着工具列表、权限状态、消息视图（本轮）、取消信号、MCP 客户端等。工具执行、权限判断、hook 都从这里读取和写入状态。
