# Claude Code 源码业务流电子书

## 这套书解决什么问题
- 以 `claude-code-sourcemap-main` 仓库中基于 `@anthropic-ai/claude-code` 2.1.88 还原出的 `restored-src/src` 源码为分析基点，厘清 Claude Code 一次请求的从入口到工具编排再到远程协同的闭环。
- 侧重“业务流程+源码映射+复刻抽象”，避免仅堆砌文件注释，而是向读者输出系统运行机理与复刻思路。
- 定位于学习型电子书：既能顺序阅读搭起主线，也提供按主题查阅的分卷/附录索引，兼顾“读源码”与“做笔记”的双重需求。

## 你会学到什么
- Claude Code 的主业务流是如何从 CLI 输入一路驱动至模型、工具、结果回填的。
- 扩展能力是如何通过 MCP、Skills、Plugins 等机制接入，同时保持主链路稳定的。
- 远程协同与后台会话/任务托管的调度策略，以及 Bridge/Remote 等系统角色如何协作。
- 如何结合源码中的关键文件与类型抽象，把理解转化为自己的多代理/工具编排设计。

## 按目标学习
- 第一次接触 Agent 开发：从 [00 前置知识](00-前置知识-从API调用到Agent.md) 开始，再顺读 Part 1 的 `03-06`，建立主链路认知。
- 想先复刻单代理 CLI：从 [01 系统全景与学习路线](01-系统全景与学习路线.md) 开始，顺读 Part 1 的 `01-06`，再读 [92 关键类型与核心抽象](part-4-附录/92-关键类型与核心抽象.md)。
- 想做扩展平台：先完成 Part 1，再顺读 Part 2 的 `07-10`，最后配合 [99 每章练习题与复刻建议](part-4-附录/99-每章练习题与复刻建议.md) 设计自己的扩展协议。
- 想做远程协同与多代理：先抓住 Part 1 主循环，再顺读 Part 3 的 `11-14`，同时配合 [90 源码地图](part-4-附录/90-源码地图-按目录反查系统能力.md) 和 [91 核心文件索引](part-4-附录/91-核心文件索引.md) 做源码反查。
- 想按源码快速回查：先看 [90 源码地图](part-4-附录/90-源码地图-按目录反查系统能力.md)，再看 [91 核心文件索引](part-4-附录/91-核心文件索引.md)，最后用各章的“源码追踪提示”回到具体文件。

## 如何配合源码一起阅读
- 以 `restored-src/src` 为主阅读对象，章节中将按业务维度标注“关键文件 + 责任 + 复刻建议”，并辅以 `README.md`、`query.ts` 等主入口的源码路径。  
- 关键章节会附带“练习题 → 复刻轨迹”提示，建议读者边读边回到源码目录定位相应结构，逐章积累系统理解。  
- 参考本项目的写作规范与章节模板，保持统一风格与可查性，后续可对照设计文档获取更详尽模板细节。
