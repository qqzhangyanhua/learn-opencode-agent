# Phase 3 Plan 01

## Objective

建立统一章节模板组件和最小必要的数据桥接，让章节页可以稳定展示学习目标、前置知识和行动闭环。

## Tasks

### Task 1: 建立章节学习摘要组件

**Files:**
- Create: `.vitepress/theme/components/ChapterLearningGuide.vue`
- Modify: `.vitepress/theme/components/types.ts`
- Modify: `.vitepress/theme/index.ts`

- [ ] Step 1: 创建 `ChapterLearningGuide.vue`，优先从当前页 frontmatter 读取 `learningGoals`、`prerequisites`、`estimatedTime`、`entryMode`、`roleDescription`
- [ ] Step 2: 为组件定义适用于理论 / 实践 / 中级三类页面的展示结构
- [ ] Step 3: 在 `types.ts` 中补齐组件 props 类型
- [ ] Step 4: 在 `.vitepress/theme/index.ts` 注册组件
- [ ] Step 5: 运行 `bun run build:strict`

### Task 2: 建立章节行动闭环组件与校验

**Files:**
- Create: `.vitepress/theme/components/ChapterActionPanel.vue`
- Modify: `package.json`
- Create: `scripts/check-chapter-experience.mjs`

- [ ] Step 1: 创建 `ChapterActionPanel.vue`，支持渲染推荐下一步、跨板块入口和行动任务
- [ ] Step 2: 让组件复用 `recommendedNext` 与 `practiceLinks`，并允许页面传入轻量 `actionItems`
- [ ] Step 3: 添加 `check:chapter-experience` 脚本并纳入 `build:strict`
- [ ] Step 4: 在校验脚本中约束首批 7 个章节必须接入统一组件
- [ ] Step 5: 运行 `bun run build:strict`

## Acceptance Criteria

- 主题中存在统一的 `ChapterLearningGuide` 与 `ChapterActionPanel`
- `build:strict` 已包含章节体验校验
- `bun run build:strict` 通过
