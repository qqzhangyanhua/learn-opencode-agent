# Phase 5 Plan 01

## Objective

先建立 Phase 5 的统一 discovery 数据层和搜索增强基础，让后续 `/discover` 页面、搜索语义和推荐关系都建立在同一份编排真源上。

## Tasks

### Task 1: 建立 discovery 编排层与统一类型语言

**Files:**
- Modify: `.vitepress/theme/data/content-meta.ts`
- Modify: `.vitepress/theme/data/content-index.data.ts`
- Create: `.vitepress/theme/data/discovery-content.ts`
- Modify: `.vitepress/theme/components/types.ts`

- [ ] Step 1: 在现有 metadata 之上补充面向前台展示的内容类型标签与 discovery 关系类型，而不是另起一套内容模型
- [ ] Step 2: 新建 `discovery-content.ts`，把学习目标、主题聚合、推荐起点、下一步和前置补课组织成统一 view model
- [ ] Step 3: 让 `content-index.data.ts` 能暴露 discovery 层需要的最小索引字段，避免页面和组件各自重复拼装
- [ ] Step 4: 在 `types.ts` 中补齐 Discovery 组件和推荐卡片所需类型
- [ ] Step 5: 运行 `bun run typecheck`

### Task 2: 增强本地搜索索引与 Phase 5 校验

**Files:**
- Modify: `.vitepress/config.mts`
- Modify: `package.json`
- Create: `scripts/check-discovery-experience.mjs`

- [ ] Step 1: 基于 VitePress local search 的 `_render` 或分段能力，把内容类型、导航提示、主题标签等语义注入索引内容
- [ ] Step 2: 新增 `check:discovery-experience` 并接入 `build:strict`
- [ ] Step 3: 在校验脚本中先验证 discovery 数据层、搜索配置和最小类型语义字段已经到位
- [ ] Step 4: 运行 `bun run build:strict`

## Acceptance Criteria

- discovery 编排层已经存在，并由既有 metadata、路径数据和实践数据构建
- 本地搜索已具备内容类型和最小学习语义增强，而不是纯页面标题检索
- `check:discovery-experience` 已纳入 `build:strict`
- `bun run build:strict` 通过
