# Phase 2 Plan 03

## Objective

让首页之外的关键入口页都具备统一的“当前位置 + 下一步”表达，降低用户脱离首页后的迷路概率。

## Tasks

### Task 1: 建立统一入口提示组件

**Files:**
- Create: `.vitepress/theme/components/EntryContextBanner.vue`
- Modify: `.vitepress/theme/index.ts`

- [ ] Step 1: 创建 `EntryContextBanner.vue`，接收板块名、定位说明、推荐下一步、辅助入口
- [ ] Step 2: 为组件设计桌面 / 移动端都清晰的卡片布局
- [ ] Step 3: 在主题入口注册组件
- [ ] Step 4: 运行 `bun run build:strict`

### Task 2: 接入关键入口页

**Files:**
- Modify: `docs/reading-map.md`
- Modify: `docs/practice/index.md`
- Modify: `docs/intermediate/index.md`
- Modify: `docs/index.md`

- [ ] Step 1: 在阅读地图页加入“当前位置 / 推荐下一步”统一提示
- [ ] Step 2: 在实践页加入统一提示，强调起点、前置与下一跳
- [ ] Step 3: 在中级篇加入统一提示，强调它是 bridge / 专题入口
- [ ] Step 4: 如有必要，在首页增加同一套结构的轻量版本，保持表达统一
- [ ] Step 5: 运行 `bun run build:strict`

## Acceptance Criteria

- 关键入口页都出现统一 banner 或结构一致的上下文提示
- 用户在这些页面能明确知道当前板块与下一步入口
- `bun run build:strict` 通过
