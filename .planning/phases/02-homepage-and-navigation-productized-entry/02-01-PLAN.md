# Phase 2 Plan 01

## Objective

把首页从“说明型文档页”升级为真正的起步页，让新用户打开首页就能立即判断自己该走哪条路。

## Tasks

### Task 1: 新增首页起步分流组件

**Files:**
- Create: `.vitepress/theme/components/HomeStartPanel.vue`
- Modify: `.vitepress/theme/index.ts`

- [ ] Step 1: 创建 `HomeStartPanel.vue`，消费 `learningPaths` 与 `sectionById`
- [ ] Step 2: 输出 3 张起步卡，分别对应“先看源码 / 先做项目 / 先补工程判断”
- [ ] Step 3: 每张卡都包含目标说明、适合人群、推荐起点和 CTA
- [ ] Step 4: 在 `.vitepress/theme/index.ts` 注册组件
- [ ] Step 5: 运行 `bun run build:strict`

### Task 2: 重构首页信息架构

**Files:**
- Modify: `docs/index.md`
- Modify: `.vitepress/theme/components/PracticePreview.vue`
- Modify: `.vitepress/theme/custom.css`

- [ ] Step 1: 在首页 hero / 首屏后插入 `<HomeStartPanel />`
- [ ] Step 2: 调整首页区块顺序，让“起步选择”早于长篇解释
- [ ] Step 3: 让 `PracticePreview.vue` 与首页首屏起步表达统一
- [ ] Step 4: 在 `custom.css` 中补齐首页分流区所需样式
- [ ] Step 5: 运行 `bun run build:strict`

## Acceptance Criteria

- `docs/index.md` 存在 `HomeStartPanel`
- 首页首屏后 1 屏内能看到至少 3 个清晰起步入口
- `bun run build:strict` 通过
