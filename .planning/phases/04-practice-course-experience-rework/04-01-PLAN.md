# Phase 4 Plan 01

## Objective

把实践篇首页从“文档总表”升级为“可选起点的课程入口”，并先建立后续项目页复用的统一数据真源。

## Tasks

### Task 1: 建立统一实践项目注册表

**Files:**
- Create: `.vitepress/theme/data/practice-projects.ts`
- Modify: `.vitepress/theme/components/types.ts`

- [ ] Step 1: 为 `P1-P23` 定义统一项目元数据，覆盖阶段、难度、时间、学习目标、前置、运行入口、完成判定和推荐下一步
- [ ] Step 2: 为实践首页和桥接组件整理“按目标开始”的推荐路线配置
- [ ] Step 3: 在 `types.ts` 中补齐相关组件所需类型
- [ ] Step 4: 写入最小必要的导出方法，供首页和项目页按 `projectId` 查询
- [ ] Step 5: 运行 `bun run build:strict`

### Task 2: 重构实践首页课程入口

**Files:**
- Create: `.vitepress/theme/components/PracticeRouteExplorer.vue`
- Create: `.vitepress/theme/components/PracticeProjectSyllabus.vue`
- Modify: `.vitepress/theme/index.ts`
- Modify: `docs/practice/index.md`
- Modify: `package.json`
- Create: `scripts/check-practice-course-experience.mjs`

- [ ] Step 1: 创建“按目标开始”入口组件，强化推荐起点而不是只给一堆链接
- [ ] Step 2: 创建“按阶段浏览”课程大纲组件，显示每阶段目标、代表项目和下一步
- [ ] Step 3: 重写 `docs/practice/index.md` 首屏和课程阶段区，让运行索引退居次级
- [ ] Step 4: 接入 `check:practice-course-experience` 并纳入 `build:strict`
- [ ] Step 5: 运行 `bun run build:strict`

## Acceptance Criteria

- 实践首页首屏存在清晰的“按目标开始”入口
- 实践首页存在课程化的阶段浏览结构
- Phase 4 自动校验已接入 `build:strict`
- `bun run build:strict` 通过
