# Phase 6 Plan 02

## Objective

把本地学习进度能力真正接入章节页与实践项目页的 guide 骨架中，并补齐自动校验，确保三态记录成为稳定的内容页体验，而不是临时附加工具条。

## Tasks

### Task 1: 接入章节页与实践页 guide 骨架

**Files:**
- Modify: `.vitepress/theme/components/ChapterLearningGuide.vue`
- Modify: `.vitepress/theme/components/PracticeProjectGuide.vue`

- [ ] Step 1: 在 `ChapterLearningGuide.vue` 中基于现有 frontmatter / metadata 解析稳定 `contentId` 与内容类型，并在 guide 顶部摘要区接入 `LearningProgressToggle`
- [ ] Step 2: 在 `PracticeProjectGuide.vue` 中为实践项目映射稳定内容标识，在不改变项目目标、前置和运行入口结构的前提下接入同一控件
- [ ] Step 3: 缺少合法标识时直接隐藏控件，不写入匿名状态，也不让错误打断页面阅读
- [ ] Step 4: 调整 guide 样式，让新增控件在桌面端和移动端都保持与现有卡片结构一致
- [ ] Step 5: 运行 `bun run build:strict`

### Task 2: 补齐进度体验的结构校验与回归约束

**Files:**
- Modify: `scripts/check-learning-progress.mjs`
- Modify: `scripts/check-chapter-experience.mjs`
- Modify: `scripts/check-practice-course-experience.mjs`

- [ ] Step 1: 扩展 `check-learning-progress.mjs`，验证 guide 组件已真正接入进度控件、统一 store 与必要状态类型
- [ ] Step 2: 更新章节体验校验，确保关键章节页继续保留学习导览与行动闭环，同时新的进度能力没有绕过 guide 层
- [ ] Step 3: 更新实践篇课程化校验，确保实践项目页继续保留项目导览与行动面板，并且 guide 内已拥有进度能力
- [ ] Step 4: 运行 `bun run build:strict`

## Acceptance Criteria

- `ChapterLearningGuide` 与 `PracticeProjectGuide` 都已内嵌统一进度控件
- 进度能力只依附 guide 层，不侵入 ActionPanel 或首页入口
- 自动校验已能覆盖进度接线与既有章节 / 实践体验不回退
- `bun run build:strict` 通过
