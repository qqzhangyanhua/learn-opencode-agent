# Phase 6 Plan 01

## Objective

先建立 Phase 6 的本地进度底层，包括统一状态模型、安全存储能力和可复用的三态切换组件，为章节页与实践页接入打下稳定基础。

## Tasks

### Task 1: 建立本地学习进度 store 与状态合同

**Files:**
- Modify: `.vitepress/theme/components/types.ts`
- Create: `.vitepress/theme/components/learning-progress/learningProgressStorage.ts`

- [ ] Step 1: 在 `types.ts` 中补齐学习进度相关的稳定类型，包括状态枚举、前台标签映射和最小记录结构
- [ ] Step 2: 新建 `learningProgressStorage.ts`，沿用现有安全 localStorage 模式，封装读取、写入、覆盖、查询和 JSON 损坏回退逻辑
- [ ] Step 3: 使用单一存储 key 维护 `contentId -> progressRecord` 映射，并在写入时补充 `updatedAt`
- [ ] Step 4: 明确无效 `contentId`、浏览器端不可用和解析失败时的默认返回值，避免把异常传播到页面层
- [ ] Step 5: 运行 `bun run typecheck`

### Task 2: 创建统一进度切换组件并接入自动校验

**Files:**
- Create: `.vitepress/theme/components/LearningProgressToggle.vue`
- Modify: `package.json`
- Create: `scripts/check-learning-progress.mjs`

- [ ] Step 1: 创建 `LearningProgressToggle.vue`，输出三种互斥状态按钮、当前状态高亮和轻量“已保存到本地”反馈
- [ ] Step 2: 让组件只依赖 `contentId`、`contentType` 和统一 store，不在组件内部再发明第二套内容识别逻辑
- [ ] Step 3: 新增 `check:learning-progress` 并接入 `build:strict`
- [ ] Step 4: 在 `check-learning-progress.mjs` 中先校验 store、切换组件和脚本接线已经到位
- [ ] Step 5: 运行 `bun run build:strict`

## Acceptance Criteria

- 已存在统一学习进度 store，而不是散落在 guide 组件里的临时状态逻辑
- 已存在统一三态切换组件，并以单一存储 key 保存状态
- `check:learning-progress` 已纳入 `build:strict`
- `bun run build:strict` 通过
