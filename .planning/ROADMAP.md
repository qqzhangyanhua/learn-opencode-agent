# Roadmap: AI Agent 产品化学习站

## Overview

这个路线图的目标不是继续线性扩写文档，而是把现有内容资产重组为一个更像课程产品的学习站。整体顺序遵循“先定义学习骨架，再优化入口与章节，再把实践做成闭环，先收口发现体验，再补最小学习进度层”的原则，这样每一阶段都能为下一阶段提供稳定基础。

## Phase Details

### Phase 1: 学习路径与元数据基础
**Goal**: 建立可复用的学习路径、内容分类和章节元数据体系，为后续首页、章节页和实践页改造提供统一数据源
**Depends on**: Nothing (first phase)
**Requirements**: [IA-02, IA-03, DISC-02]
**Success Criteria** (what must be TRUE):
  1. 用户可以访问按目标组织的学习路径页，并看到清晰的推荐阅读顺序
  2. 用户可以明确区分理论篇、实践篇和中级篇的角色及进入方式
  3. 站点已有可复用的章节/实践元数据来源，后续页面不必继续完全手写维护
**Plans**: 3 plans

Plans:
- [x] 01-01: 定义学习路径、内容类型和章节元数据模型
- [x] 01-02: 建立路径页与元数据聚合机制
- [x] 01-03: 将现有核心内容接入新的路径与分类体系

### Phase 2: 首页与导航产品化入口
**Goal**: 让用户在首页和全站导航中更快找到适合自己的入口，并始终知道自己所在的学习板块
**Depends on**: Phase 1
**Requirements**: [IA-01, HOME-01, HOME-02, HOME-03]
**Success Criteria** (what must be TRUE):
  1. 新用户进入首页后，可以在短时间内识别适合自己的起步路线
  2. 用户可以从全站导航快速进入核心学习路径、实践入口和辅助页面
  3. 用户在主要页面中能理解当前板块与下一层可进入入口
**Plans**: 3 plans

Plans:
- [x] 02-01: 重构首页信息架构与角色分流区
- [x] 02-02: 重构导航、侧边栏与板块入口逻辑
- [x] 02-03: 建立跨页面一致的当前位置与下一步入口提示

### Phase 3: 章节学习体验标准化
**Goal**: 为核心章节建立统一的学习结构，让用户在阅读时明确知道“为什么读、读之前需要什么、读完做什么”
**Depends on**: Phase 2
**Requirements**: [CHAP-01, CHAP-02, CHAP-03, CHAP-04]
**Success Criteria** (what must be TRUE):
  1. 用户打开核心章节时能看到学习目标、适合人群和预计投入时间
  2. 用户打开核心章节时能看到前置知识或推荐先读章节
  3. 用户读完核心章节后能看到明确的下一步推荐和对应练习/实践入口
**Plans**: 3 plans

Plans:
- [x] 03-01: 设计统一章节模板与可复用组件
- [x] 03-02: 为关键章节补齐目标、前置与行动闭环信息
- [x] 03-03: 建立章节与理论/实践/专题之间的推荐关系

### Phase 4: 实践篇课程化重构
**Goal**: 把实践篇从“脚本与文章列表”升级为更像课程平台的跟练体验
**Depends on**: Phase 3
**Requirements**: [PRAC-01, PRAC-02, PRAC-03]
**Success Criteria** (what must be TRUE):
  1. 用户可以按学习阶段或目标浏览实践项目
  2. 用户在每个实践项目页面都能看到目标、前置、运行入口和完成标准
  3. 用户可以从相关理论章节直接进入对应实践项目，形成看完就练的闭环
**Plans**: 3 plans

Plans:
- [x] 04-01: 重构实践篇索引与阶段化浏览入口
- [x] 04-02: 标准化实践项目页面信息结构
- [x] 04-03: 建立理论章节到实践项目的双向关联

### Phase 5: 搜索与发现体验收口
**Goal**: 让用户通过搜索和发现机制更快找到正确内容，并能区分不同内容类型
**Depends on**: Phase 4
**Requirements**: [DISC-01, DISC-03]
**Success Criteria** (what must be TRUE):
  1. 用户可以通过搜索或发现入口快速定位某个主题对应的章节、实践项目或专题页
  2. 用户在搜索或导航结果中可以区分内容类型
  3. 全站的内容发现体验与新的学习路径结构保持一致
**Plans**: 2 plans

Plans:
- [x] 05-01: 优化搜索索引内容与结果展示结构
- [x] 05-02: 收口全站发现入口、推荐关系和内容类型标识

### Phase 6: 本地学习进度 MVP
**Goal**: 让用户在章节页与实践项目页中手动记录本地学习状态，形成最小可用的进度感
**Depends on**: Phase 5
**Requirements**: [PROG-01, PROG-02]
**Success Criteria** (what must be TRUE):
  1. 用户在关键章节页与实践项目页的 guide 区域可以看到统一进度控件
  2. 用户可以把单个内容标记为 `稍后再看`、`从这里继续` 或 `已完成`，且三态互斥
  3. 页面刷新后状态仍然保留在本地，本地存储异常时正文与课程骨架仍可正常使用
**Plans**: 2 plans

Plans:
- [ ] 06-01: 建立本地进度 store 与统一状态切换组件
- [ ] 06-02: 接入章节页 / 实践页学习骨架并补齐校验与降级处理
