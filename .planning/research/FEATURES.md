# 产品化 AI Agent 学习站研究：功能特征

**研究日期：** 2026-03-21
**项目阶段：** Subsequent milestone
**研究问题：** 产品化的技术学习站，哪些能力是“用户期望的基本盘”，哪些是差异化增强？哪些反而应该明确不做？

## 结论摘要

对你这个项目来说，最重要的不是功能越多越像产品，而是让用户形成**清晰路径、即时定位、低摩擦进入实践**。结合当前目标用户和 MDN 的课程化设计方式，v1 应把“学习路径、目标感、练习闭环、推荐下一步”视为 table stakes。

MDN Curriculum 和教育者资源的共同特点是：
- 明确按模块组织内容
- 给不同起点的人“从哪里开始”的答案
- 每个学习单元都带学习目标
- 路径中有测试、挑战或项目，防止只读不练

这和你项目当前缺口高度一致。

## Table Stakes（v1 必须有，否则用户会流失）

### 1. 明确的起步入口

**用户预期：**
- 进入首页后能迅速知道自己该走哪条路径
- 不需要先理解整棵目录树才能开始

**在本项目中的具体化：**
- 首页显著区分 2-3 条核心路线，例如：
  - 从零入门路线
  - 工程化进阶路线
  - 边学边做路线

**复杂度：** 中  
**依赖：** 首页改版、元数据梳理、路径页

### 2. 章节级学习目标

**依据：**
- MDN 教育者页面明确提到，每篇文章顶部应有学习成果（learning outcomes）

**在本项目中的具体化：**
- 每章显示：
  - 你将学会什么
  - 需要先会什么
  - 适合什么阶段读
  - 预计阅读/动手时间

**复杂度：** 中  
**依赖：** frontmatter 结构统一

### 3. 清晰的学习路径

**依据：**
- MDN 明确强调要给学生 clear progression、clear learning objectives、regular assessments

**在本项目中的具体化：**
- 不再只按目录顺序读
- 需要能从“目标”出发选择路线
- 每一页都能告诉用户“上一站 / 当前 / 下一站”

**复杂度：** 中高  
**依赖：** 路径模型、导航重构、章节元数据

### 4. 练习与挑战闭环

**依据：**
- MDN Core modules 配有 “Test your skills” 与 “Challenge” 文章
- 教育者指南推荐理论测试 + 实战项目结合

**在本项目中的具体化：**
- 每章至少给出：
  - 一个最小行动项
  - 对应实践入口或延伸练习
  - 推荐继续做的项目

**复杂度：** 中  
**依赖：** 章节模板统一、practice 索引

### 5. 更像课程平台的实践篇入口

**用户预期：**
- 实践篇不是“脚本清单”，而是“我应该怎么跟练”

**在本项目中的具体化：**
- 实践页按阶段/目标组织
- 每个实践项目有：
  - 学习目的
  - 前置章节
  - 运行命令
  - 完成标准
  - 下一练建议

**复杂度：** 中高  
**依赖：** practice 元数据、实践列表重构

## Differentiators（有竞争力的增强项）

### 1. 按用户目标而不是按内容分类组织

**为什么重要：**
- 大多数文档站按作者结构组织
- 你这个站的差异化机会，是按“用户为什么来”来组织

**示例：**
- 我想搞懂 AI Agent 原理
- 我想搭出一个最小 Agent
- 我想补工程化与生产化

### 2. 理论、实践、中级篇之间的联动推荐

**为什么重要：**
- 这是你现有内容资产的最大价值来源
- 很多站点要么只有概念，要么只有 demo；你同时有三条线

**具体形式：**
- 每章文末推荐“理论补充”“立即动手”“进阶专题”

### 3. 章节与实践的一一对应视图

**为什么重要：**
- 对想直接抄作业的人非常友好
- 能把“阅读理解”转成“立即行动”

## Anti-Features（刻意不做）

### 1. 用户登录 / 云端进度同步

**为什么不做：**
- 与 v1 目标不匹配
- 会把静态站问题变成账号系统问题

### 2. 社区评论 / 讨论区

**为什么不做：**
- 这是运营系统，不是学习路径系统
- 会分散对内容产品化的投入

### 3. AI 助教 / 问答

**为什么不做：**
- 不是不能做，而是当前最重要的问题不是“站里没有 AI”，而是“学习路径还不够清晰”

### 4. 题库评分系统

**为什么不做：**
- 现在更需要“行动建议 + 跟练闭环”，不需要复杂判题

## 对需求定义的直接建议

### v1 必须覆盖的功能类目

- 首页起步入口
- 用户路线页 / 学习路径页
- 章节统一元信息模块
- 实践项目课程化卡片与索引页
- 章节末尾下一步推荐
- 搜索与导航的课程化优化

### v2 可以延后的功能类目

- 本地进度保存
- 收藏 / 稍后阅读
- 个性化推荐
- 课程完成仪表盘

## 置信度

- **高**：起步入口、路径、学习目标、练习闭环是 table stakes
- **高**：实践篇课程化是这个项目的核心增长点
- **高**：登录、AI 助教、社区等不应进入 v1
