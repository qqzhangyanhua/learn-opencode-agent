混合方案 — 动画技术分配

你选择了混合方案。现在需要决定：哪些动画用 CSS，哪些用 Lottie？

分配原则

  • CSS 适合：状态切换、流程演示、简单过渡、需要与数据绑定
  • Lottie 适合：复杂路径、精细动效、视觉冲击力、独立播放

5 个动画的复杂度分析

1. 什么是 Agent

内容：LLM → Agent 的演进（单次调用 → 循环执行）

复杂度：⭐⭐⭐ 中等

建议：CSS

理由：主要是状态切换和流程演示，CSS 足够

2. 工具调用

内容：Function Calling 完整流程（注册 → 解析 → 执行 → 返回）

复杂度：⭐⭐⭐⭐ 较高

建议：Lottie

理由：涉及数据流动、参数解析等复杂路径，Lottie 更直观

3. 多轮对话

内容：上下文在多轮对话中的传递和累积

复杂度：⭐⭐ 简单

建议：CSS

理由：消息累积动画，类似 MessageAccumulator，CSS 即可

4. 多 Agent 协作

内容：Agent 间消息传递、任务分配、结果聚合

复杂度：⭐⭐⭐⭐⭐ 最高

建议：Lottie

理由：多个 Agent 并行、消息路由、状态同步，视觉复杂度高

5. 记忆系统

内容：短期记忆 → 长期记忆转换，向量检索

复杂度：⭐⭐⭐⭐ 较高

建议：Lottie

理由：涉及向量空间、相似度计算等抽象概念，需要精细视觉表达

  • 多轮对话
