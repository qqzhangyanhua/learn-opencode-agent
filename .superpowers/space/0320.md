一、需求概述

1.1 背景

当前电子书站点已有 27 个交互组件，但缺少帮助新手快速理解核心概念的动画演示。需要新增 5 个核心概念动画，降低学习门槛。

1.2 目标

- 主要目标： 通过动画帮助新手在 5 分钟内理解 AI Agent 核心概念
- 次要目标： 提升电子书的视觉吸引力和用户体验
  1.3 核心动画清单  


1. 什么是 Agent — LLM → Agent 的演进过程
2. 工具调用 — Function Calling 完整流程
3. 多轮对话 — 上下文传递机制
4. 多 Agent 协作 — Agent 间消息传递与任务分配
5. 记忆系统 — 短期 → 长期记忆转换  


1.4 技术决策

- 实现方式： 混合方案（CSS + Lottie）
  - CSS 动画：什么是 Agent、多轮对话（2个）
  - Lottie 动画：工具调用、多 Agent 协作、记忆系统（3个）
- 交互模式： 滚动触发（进入视口 30% 时自动播放）
- 开发策略： 并行开发（2-3周完成）

三、核心基础设施

3.1 滚动触发 Hook

文件： animations/core/useIntersectionObserver.ts

功能： 监听元素进入视口，触发动画播放

接口设计：  
 export interface UseIntersectionObserverOptions {  
 threshold?: number // 触发阈值，默认 0.3  
 rootMargin?: string // 根边距，默认 '0px'  
 triggerOnce?: boolean // 是否只触发一次，默认 true  
 }

export function useIntersectionObserver(  
 target: Ref<HTMLElement | null>,  
 options?: UseIntersectionObserverOptions  
 ): {  
 isVisible: Ref<boolean>  
 hasTriggered: Ref<boolean>  
 }

实现要点：

- 使用 Intersection Observer API
- 支持 triggerOnce 模式（性能优化）
- 返回 isVisible 和 hasTriggered 两个状态
- 组件卸载时自动清理 observer
  3.2 动画容器组件  


文件： animations/core/AnimationContainer.vue

功能： 统一的动画容器，提供滚动触发、控制按钮、状态显示

Props：  
 interface AnimationContainerProps {  
 title?: string // 动画标题  
 statusText?: string // 状态文本  
 }

Slots：  
 default: {  
 isVisible: boolean // 是否可见  
 restart: () => void // 重新播放函数  
 }

样式特性：

- 容器淡入动画（opacity + translateY）
- 统一的边框、圆角、背景色
- 响应式布局
- 支持亮色/暗色模式
  3.3 Lottie 播放器组件  


文件： animations/core/LottiePlayer.vue

功能： 封装 lottie-web，提供统一的 Lottie 动画播放接口

Props：  
 interface LottiePlayerProps {  
 animationData: object // Lottie JSON 数据  
 autoplay?: boolean // 自动播放，默认 false  
 loop?: boolean // 循环播放，默认 false  
 speed?: number // 播放速度，默认 1  
 }

Events：  
 complete: [] // 动画完成  
 loopComplete: [] // 循环完成

暴露方法：  
 play(): void // 播放  
 pause(): void // 暂停  
 stop(): void // 停止  
 restart(): void // 重新播放

实现要点：

- 使用 SVG 渲染器（性能更好）
- 组件卸载时销毁 Lottie 实例
- 支持响应式控制（watch autoplay）

四、CSS 动画设计

4.1 什么是 Agent

文件： animations/css/WhatIsAgent.vue

动画内容：

阶段 1：传统 LLM（0-2s）  
 输入 → LLM → 输出

- 展示单次调用模式
- 标注"一次性调用，无法处理复杂任务"  


阶段 2：AI Agent（2-5s）

- 展示循环执行模式
- 高亮循环箭头
- 标注"循环执行，直到任务完成"

样式规范：

- 节点颜色：品牌色（Teal）渐变
- 字体：14px，600 weight
- 间距：节点间距 16px
- 动画时长：每个阶段 2s

五、Lottie 动画设计

5.1 工具调用

文件： animations/lottie/FunctionCalling.vue

动画内容：

关键帧设计：

1. 0-3s：Agent 发起请求


    - Agent 图标出现
    - JSON 数据流动（粒子效果）
    - 显示工具名称和参数

2. 3-8s：参数解析


    - JSON 数据展开
    - 参数验证（✓ 标记）
    - 结构化数据形成

3. 8-15s：工具执行


    - 齿轮转动动画
    - 进度条填充
    - 工具图标闪烁

4. 15-20s：结果返回


    - 数据流回 Agent
    - 结果高亮显示
    - Agent 图标确认


Lottie 设计要求（给设计师）：

- 时长：15-20秒
- 尺寸：800x400px
- 风格：扁平化，使用品牌色（#0d9488）
- 格式：JSON（压缩后 < 50KB）
- 帧率：30fps  


Vue 组件实现：  
 <template>  
 <AnimationContainer title="工具调用流程" @restart="handleRestart">  
 <template #default="{ isVisible }">  
 <LottiePlayer  
 ref="playerRef"  
 :animation-data="animationData"  
 :autoplay="isVisible"  
 :loop="false"  
 @complete="handleComplete"  
 />  
 </template>  
 </AnimationContainer>  
 </template>

  <script setup lang="ts">                                                                                                                    
  import { ref } from 'vue'                                                                                                                   
  import AnimationContainer from '../core/AnimationContainer.vue'                  
  import LottiePlayer from '../core/LottiePlayer.vue'                                                                                         
  import animationData from './function-calling.json'                                                                                         
                                                                                                                                              
  const playerRef = ref<InstanceType<typeof LottiePlayer> | null>(null)                                                                       
                                                                                                                                              
  function handleRestart() {                                                                                                                  
    playerRef.value?.restart()                                                                                                                
  }                                                                                                                                           
                                                                                                                                              
  function handleComplete() {                                                                                                                 
    console.log('Animation completed')                                             
  }                                                                                                                                           
  </script>

八、性能优化

8.1 包体积优化

策略：

1. Lottie JSON 按需加载  
   const animationData = await import('./function-calling.json')
2. 代码分割


    - 利用 VitePress 的自动代码分割
    - 动画组件独立打包

3. Lottie JSON 压缩


    - 移除不必要的关键帧
    - 使用 Lottie 压缩工具
    - 目标：每个 < 50KB


预期效果：

- 首屏加载：不增加（动画按需加载）
- 动画页面：+50KB（lottie-web）+ 50KB（单个 JSON）
  8.2 运行时性能  


滚动触发优化：

- triggerOnce: true — 只触发一次
- threshold: 0.3 — 30% 可见时触发
- 避免同时播放多个动画  


Lottie 优化：

- 使用 SVG 渲染器（而非 Canvas）
- 动画完成后销毁实例
- 限制同时播放数量（最多 1 个）  


CSS 动画优化：

- 使用 transform 和 opacity（GPU 加速）
- 避免 layout thrashing
- 使用 will-change 提示浏览器  


目标指标：

- Lighthouse Performance：> 90
- FPS：> 60
- 内存占用：< 50MB

十、风险和缓解

风险 1：设计师延期

影响： Lottie 动画无法按时交付

概率： 中等

缓解策略：

- Week 1 先完成 CSS 动画，Week 2 再等 Lottie
- 提前与设计师对齐需求和时间表
- 准备备选方案（CSS 实现简化版）  


风险 2：Lottie 文件过大

影响： 包体积超标，影响加载速度

概率： 中等

缓解策略：

- 限制每个动画 < 50KB
- 使用 Lottie 压缩工具
- 移除不必要的关键帧和图层
- 降低帧率（30fps → 24fps）  


风险 3：浏览器兼容性问题

影响： 部分浏览器动画异常

概率： 低

缓解策略：

- 使用 Intersection Observer polyfill
- Lottie 使用 SVG 渲染器（兼容性最好）
- 提供降级方案（静态图 + 文字说明）  


风险 4：性能问题

影响： 动画卡顿，影响用户体验

概率： 低

缓解策略：

- 限制同时播放的动画数量
- 使用 GPU 加速（transform、opacity）
- 动画完成后销毁实例
- 移动端降低动画复杂度
